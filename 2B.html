This generative sketch forms a face using simple geometric shapes. The top-left rectangle and top-right triangle represent eyes that shift position and color in response to mouse movement.

Adapted from a p5.js symmetry drawing example, this sketch introduces dynamic color and stroke thickness based on mouse position. The original radial symmetry logic remains, but the lines now shift in hue and weight depending on cursor location, creating a responsive, kaleidoscopic drawing experience unique to each interaction.

Based on the p5.js example “Animation with Events,” this sketch replaces the bouncing ball with a column of moving circles influenced by mouse position. Color shifts dynamically using HSB mode, and clicking toggles animation while displaying a fading farewell message during the first interaction.

Inspired by the “Wave” section from compform.net/sound, this sketch uses p5.FFT to visualize live microphone input. The waveform is drawn as a field of ellipses, with map() translating audio values into screen coordinates — a technique that mimics sensitivity and taught me how input ranges can be dynamically scaled for visual feedback.

This is the voice recognition sketch by Karen Ann, this code uses p5.SpeechRec to transcribe live speech into text. While I don't fully understand how all parts of the speech recognition API work yet, setting it up helped me begin to understand how external scripts and libraries are linked and used in an HTML file through the index. It's a small but important step in grasping how browser-based interactivity works.
